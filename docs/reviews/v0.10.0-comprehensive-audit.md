## Minor Findings

1. **Field header caption was a single dense line** — `property_id | Requirement |
   Range | evidence hint` was hard to scan quickly. Improved to two lines: bold label
   with requirement suffix on first line, compact `·`-separated metadata on second.
   (Applied.)

2. **"Product is for sale" checkbox label was ambiguous** — the label gave no
   indication it was form-only without hovering the help tooltip. Changed label text
   to include `(form helper only — no JSON-LD emitted)` so the constraint is visible
   without hover. (Applied.)

3. **Download button in Builder was labelled "Download JSON-LD"** — identical to the
   converter download label, which could mislead a user who has both workflows open.
   Changed to "Download prototype JSON-LD" with a corresponding filename prefix
   `manual_jsonld_prototype_`. (Applied.)

4. **"BMS/XPath traceable" hero chip was unscoped** — applied to the whole app but
   refers to converter output only. The prototype Builder output is not BMS/XPath
   traceable by design. Chip relabelled "Converter: BMS/XPath traceable". (Applied.)

5. **Vocabulary status sidebar lacked "offline" qualifier** — the status panel showed
   `Local snapshot: 1.17` without clarifying it is an offline-only snapshot. Added
   `(offline)` qualifier to avoid implying live data. (Applied.)

6. **Builder section 1 heading** — the summary line did not call out that the Builder
   is not the GDSN XML converter. Added clarifying phrase. (Applied.)

7. **"Generated JSON-LD Output" section title** — renamed to "Prototype JSON-LD
   Preview" with a sub-note "This is not converter output." Removes ambiguity when
   users switch between the Convert and Prototype workflows. (Applied.)

8. **Prototype warning emoji** — added `⚠️` prefix to make the warning scannable
   without relying solely on Streamlit's yellow background. (Applied.)

## JSON-LD Correctness Findings

All correct:

- `@context` is `["https://ref.gs1.org/voc/data/gs1Voc.jsonld", {"schema": "https://schema.org/"}]` in every output.
- `@type` is `"Product"` (not `"gs1:Product"` — compact form, consistent with manual mode intent).
- GTIN generates `@id` as `https://id.gs1.org/01/{gtin}`.
- GTIN normalization strips non-digits via `re.sub(r"\D", "", ...)` — safe.
- Compact property names strip the `gs1:` prefix; `"gs1:productName"` serializes as `"productName"`.
- Empty fields are omitted — `update_builder_value(state, pid, "")` removes the key.
- Clearing a field removes it from JSON-LD (verified by new test).
- Language-tagged values produce `[{"@language": "en", "@value": "..."}]` arrays.
- Default language is applied consistently from the `default_language` state key.
- Quantity fields emit `{"value": ..., "unitCode": "..."}` objects — safe scalars only.
- Quantity value without unitCode: warns, does not emit.
- unitCode without quantity value: warns, does not emit.
- Unsupported nested ranges (e.g. `gs1:CertificationDetails`): shown as planned, not emitted.
- URL validation warns if not HTTP(S) and does not emit.
- Output is deterministic: `sorted(values)` iteration order + `sort_keys=False` on a fixed insertion order.
- Preview and download are both produced from `serialize_builder_state_to_jsonld` — consistent.
- Manual output does not use converter mapping evidence.

## Remaining Recommendations for v0.10.1 or v0.11.0

1. Fix the stale "Potential work after v0.7.0" heading in `docs/roadmap.md`.
2. Resolve or explicitly defer the 8 existing catalog `webvoc_review_required` warnings through SDR decisions.
3. Consider splitting the three-column Builder layout into a two-step flow at narrow viewports where the three columns collapse to stacked containers.
4. Consider adding a visible "fields entered: N" count in the Builder output column so users know how many properties are in the current prototype.
5. The `st.json` + `st.code` dual preview in the Builder output adds scroll depth. A future release could collapse `st.json` by default or make it opt-in via expander.
6. Add keyboard navigation between Builder groups (currently only selectbox-driven).
7. Investigate the OS-level `PermissionError` on `pytest-of-IVOLAD` temp directory to unblock the two monkeypatch-based import tests in CI environments.
