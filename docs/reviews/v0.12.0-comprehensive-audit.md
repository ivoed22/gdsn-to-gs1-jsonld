### UI/UX Findings and Improvements (applied)

**U1 — PP top warning used st.warning() rather than the custom CSS class**  
The `pp-prototype-warning` CSS class was defined in `app/ui.py` but the PP
workflow used a plain `st.warning()` call that didn't use it. Changed to
`st.markdown()` with the `pp-prototype-warning` div. Added `EU DPP regulatory
compliance` to the warning text for completeness.

**U2 — Source Inventory had no empty state when manifest not yet loaded**  
Added a styled `empty-state` div with PP marker explaining that the user should
click "Load Source Manifest".

**U3 — Inventory Summary description lacked context for missing files**  
Updated section summary to clarify that missing files are placeholder entries
requiring a separate download.

**U4 — Validation status metric label was the raw status string uppercased**  
Added a human-readable status label map: "Valid", "Invalid — review errors
below", "Schema error — cannot validate", "Not run". Also renamed the metric
headers from "Status / Errors / Warnings" to "Structural validation status /
Schema errors / Validator warnings" to be clearer about scope.

**U5 — Validation error table column header was generic "error"**  
Added a row number column `#` and kept the `error` column so the table is
easier to scan when there are multiple errors.

**U6 — Validation report download label didn't indicate structural scope**  
Download intro title changed from "Validation report JSON" to "Structural
validation report JSON". Description now includes the "Prototype/reference only
— not official GS1 validation" qualifier.

**U7 — Examples tab info text didn't distinguish committed vs placeholder files**  
Updated both the section header description and the `st.info()` call to note
that the minimal example is committed to the repository for structural testing
and that placeholder entries require a separate download.

**U8 — Example preview expander title was generic**  
Changed from "Example JSON preview" to "Example JSON preview
(prototype/reference only — not production data)" to maintain the governance
posture inside the preview area.

**U9 — Placeholder file missing message could be clearer**  
Updated from "Local file not yet present: ..." to "Local file not yet
available: `...`. This is a placeholder entry. Download the file from the URL
in the source manifest and place it at the path shown."

**Streamlit test update:**  
`test_product_passport_bridge_warning_text_appears` was updated because the PP
warning changed from a `st.warning()` element (visible via `app.warning`) to a
custom HTML `st.markdown()` element. The test now checks `app.markdown` for
prototype/reference governance keywords. 13/13 Streamlit tests pass.
