# v0.3.0 — BMS/XPath-aligned Certification & Document Mapping

## Scope

- `certificationInformationModule`
- certification details
- certification document references
- DPP-like document links from referenced file information

## Out of scope

- full digital product passport model
- Verifiable Credential verification
- DCAT or DPROD
- resolver calls or URL dereferencing
- certificate validation
- full GDSN XSD validation

## Mapping confidence

Official GDSN 3.1.36 BMS/XPath rows and locally validated GS1 Web Vocabulary
terms support the certification parent, object type, standard, value, audit
date, start date, and end date.

The following alignments remain experimental or need semantic review:

- `certificationIdentification` is a non-URI reference in the example. It is
  emitted as `schema:identifier` instead of being mislabeled as
  `gs1:certificationURI`.
- `certificateIssuanceDateTime` is not represented in the current catalog and
  is retained as experimental `schema:dateIssued`.
- `certificationOrganisationIdentifier` is a GLN. It is emitted through
  `gs1:certificationAgency` with a semantic-review warning because that
  property describes agency text, not an organisation identifier.
- document metadata uses Schema.org terms.
- `gs1:referencedDocument` is experimental and is not present in the validated
  GS1 Web Vocabulary.

No certificate verification, URL dereferencing, resolver call, or external
service is performed.
