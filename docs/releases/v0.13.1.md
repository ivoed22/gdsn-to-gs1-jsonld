# v0.13.1 — Guided Convert Workflow

## Overview

Version v0.13.1 is a patch release that presents the **Convert GDSN XML →
Single XML** path as a guided four-step flow with a progress indicator. It adds
no new features and changes no conversion behaviour — it simply reframes the
existing, real conversion pipeline as clear steps.

No mock data. No fabricated coverage. No compliance claims. This release does
not claim official GS1 validation or production compliance.

## What changed

### Guided four-step Convert flow

The Single XML conversion path now shows a progress indicator and four labelled
steps, colour-coded (teal / amber / orange / green):

1. **Upload** — choose one GDSN product XML; the active mapping profile is shown
   inline. Processing stays in memory.
2. **Mapping** — review the applied mapping profile and the source-to-property
   trace.
3. **Validate** — real validation status, the product identity (`@id`), and a
   preview of the generated GS1 Web Vocabulary JSON-LD.
4. **Export** — download the four real outputs and review guidance, or clear
   results.

The flow is wired to the real converter/validator/reporter. The same outputs are
produced and downloadable:

- product JSON-LD
- mapping report XLSX
- validation report JSON
- unmapped fields JSON

### Scope

- All seven workflows remain reachable (Convert, Explore GS1 Web Vocabulary,
  Create JSON-LD Prototype, Generate Mapping Candidates, Validate Product
  Passport Sources, Build Product Passport Prototype, Standards Review).
- Every governance warning is preserved.
- Session persistence, "Clear results", and profile-change-clears-results
  behaviour are unchanged.
