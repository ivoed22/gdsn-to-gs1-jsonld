# v0.10.0 — Manual JSON-LD Prototype Builder

## Overview

Version v0.10.0 adds a manual JSON-LD authoring workflow to the Streamlit app.
Users can select GS1 Web Vocabulary properties, enter values, and preview
prototype JSON-LD live.

This release is a prototype-authoring release. It does not change converter
logic, mapping YAML, mapping catalog data, batch conversion, sample conversion
semantics, or Web Vocabulary snapshots.

## Added

- Added `builder_manifest/product_builder_v0_10.yaml`.
- Added manifest-driven manual-builder functions in
  `src/gdsn_to_gs1_jsonld/jsonld_builder.py`.
- Added a fourth Streamlit workflow card: `Create JSON-LD Prototype`.
- Added Builder controls for root class, product category, default language,
  and thematic group selection.
- Added range-aware form fields for strings, language-tagged text, URLs,
  booleans, numbers, dates, and simple quantities.
- Added live Generated JSON-LD Output with `st.json`, copy-friendly code block,
  validation warnings, and JSON-LD download.
- Added explicit prototype/governance warning.
- Added tests for manifest loading, state defaults, input inference,
  serialization, validation warnings, deterministic output, and Streamlit
  workflow rendering.
- Added `docs/manual-jsonld-builder.md`.

## Supported In v0.10.0

- Root class: `Product`
- Compact JSON-LD property keys without the `gs1:` prefix
- `@id` generation from GTIN:
  `https://id.gs1.org/01/{gtin}`
- Language-tagged values as JSON-LD language maps
- Simple quantity objects with `value` and `unitCode`
- Safe URL link-type properties

## Preserved

- Converter logic is unchanged.
- Batch conversion behavior is unchanged.
- Single-file converter output is unchanged.
- Mapping YAML files are unchanged.
- Mapping catalog data is unchanged.
- Web Vocabulary snapshots are unchanged.
- No warnings were suppressed.
- Mapping Candidate Generator was not created.
- No online fetching or external API dependency was added.
- No large dependency was added.
