# v0.13.5 — Release Hygiene & Test Environment Fix

## Overview

Version v0.13.5 is process and developer-environment maintenance only. There is
**no user-facing behavior change**: the converter, batch behavior, single-file
output, mapping YAML, mapping catalog, Web Vocabulary snapshots, the Streamlit
workflows, and the Manual JSON-LD Builder output are all unchanged.

It is the first release cut under the tighter, **tag-per-meaningful-change**
cadence.

## Release cadence, documented

- Adds an `## Unreleased` section to `CHANGELOG.md` as the home for in-progress
  work between releases.
- Adds a "Release process" section to `docs/roadmap.md` describing the
  tag-per-change cadence: each meaningful change is cut as its own CI-gated
  annotated tag + GitHub Release, keeping "what is released" unambiguous.
- `tests/test_version_consistency.py` continues to enforce that
  `pyproject.toml`, the `app/ui.py` `APP_VERSION`, the latest CHANGELOG heading,
  the matching `docs/releases/vX.Y.Z.md`, and the README stay in sync.

## Deterministic local test runs

On machines whose default pytest temp root
(`<system temp>/pytest-of-<user>`) is not writable — for example a leftover
directory with a restrictive ACL — every `tmp_path`-using test errored out
during setup with `PermissionError: [WinError 5]` before running (~34 errors).

`tests/conftest.py` now points pytest's temporary-file root at a git-ignored,
repo-local `.pytest-tmp` directory (via `PYTEST_DEBUG_TEMPROOT`, honouring an
explicit override if the environment already set one). CI was already green and
stays green; locally the full suite now reports **`197 passed`** with no errors.
