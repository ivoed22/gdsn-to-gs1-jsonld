# v0.6.0 — Web Vocabulary Update Monitor

## Overview

Version 0.6.0 adds controlled monitoring of official GS1 Web Vocabulary and
linktype resources, local snapshot validation, mapping-catalog revalidation,
and clearer warning classification.

Normal product conversion does not access the network. Semantic mappings are
never rewritten automatically.

## Added

- Local `webvoc/current/` snapshots for JSON-LD, Turtle, linktypes, and
  metadata.
- `check-webvoc-updates` CLI command with online comparison and `--no-network`
  validation.
- Explicit `--update-snapshot` support for controlled snapshot refresh.
- `revalidate-mapping-catalog` CLI command.
- JSON and XLSX Web Vocabulary update reports.
- JSON, XLSX, and CSV mapping-catalog revalidation reports.
- Optional versioned v0.6 revalidated catalog output.
- Stable GS1 linktype recognition, including `dpp` and `certificationInfo`.
- Structured warning categories and recommended actions.
- Compact Streamlit vocabulary snapshot status.
- Tests and documentation for snapshot and revalidation workflows.

## Changed

- Existing catalog status values are recognized by quality checks.
- Governance and modelling warnings are classified more clearly.
- Source catalog and mapping YAML files remain unchanged by default.
- `check-catalog` warnings are reduced from 23 to 8.
- `check-mapping` warnings are reduced from 30 to 15.

## Compatibility

- Existing converter outputs and mapping profiles remain unchanged.
- v0.1.0, v0.2.0, v0.3.0, and v0.5 sample behavior is preserved.
- Normal CLI conversion and Streamlit conversion remain offline.

## Limitations

Unresolved nutrient, image, document/DPP, certification, and governance
warnings remain where automatic resolution would require a semantic decision.
The monitor detects evidence changes; it does not establish mapping authority
or full conformance. These remaining warnings are non-blocking governance and
modelling issues.
