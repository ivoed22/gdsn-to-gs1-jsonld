# v0.13.2 — Workspace Layout & Theme Navigation Polish

## Overview

Version v0.13.2 is a UI/UX and information-architecture polish release. It makes
the multi-workflow app feel like a guided standards workbench — primary route
first, supporting thematic tools second — without changing any conversion,
validation, or reporting behaviour.

No new features. No mock data. No fabricated coverage. No compliance claims.
This release does not claim official GS1 validation or production compliance.

## What changed

### Wider main workspace

The main container max-width is raised to ~82rem (≈1310px), giving workflow
cards, JSON previews, mapping tables, and validation reports more room without
going full-bleed. Long text stays constrained (`.app-summary`, `.trace-note`,
etc.). Streamlit `st.columns` manage their own responsive stacking; where exact
column control at ~800px is not reliably possible via CSS, the themed grouping
(max three cards per row) is the mitigation.

### Sidebar reframed as compact workspace status/context

The sidebar is no longer a control panel. It now shows:

- **Workspace status** — app version, mode (prototype / review), storage
  (in-memory), warnings (visible, not suppressed).
- **Current context** — mapping profile selector (still functional; also applies
  inside Convert GDSN XML) and the active mapping file.
- **Sources** — GS1 Web Vocabulary snapshot/version, and Product Passport
  schemas (built-in minimal schema; external placeholders not downloaded).
- **Governance** — standards-review backlog, plus "No official GS1 validation"
  and "No production compliance claim."

Supported-group chips moved into a collapsed expander.

### Themed landing navigation

The overview groups the seven workflows under headings, with Convert as the
recommended starting point:

- **Recommended path** — Convert GDSN XML.
- **Vocabulary & Mapping** — Explore GS1 Web Vocabulary, Generate Mapping
  Candidates, Standards Review.
- **JSON-LD Prototyping** — Create JSON-LD Prototype.
- **Product Passport Bridge** — Validate Product Passport Sources, Build Product
  Passport Prototype.

### Shorter card copy and clearer Product Passport distinction

Each card is now one sentence plus a short outcome line. The two Product Passport
workflows are clearly distinct: **Validate Product Passport Sources** inspects
local sources, schemas, and examples; **Build Product Passport Prototype** wraps
GS1 JSON-LD into a prototype Product Passport envelope. Both remain
minimal-schema prototype mode — not official GS1 validation, not EU DPP
regulatory compliance, not production-ready.

### Traceability rail labelled

The rail is labelled "Core conversion traceability" (XML → BMS/XPath →
GS1 JSON-LD → SDR) with a note that Product Passport workflows build on the GS1
JSON-LD output as prototype/reference tooling — not an official traceability
output.

### Compacted hero

Refreshed hero copy and badges (In-memory, BMS/XPath traceable, Review-only,
Prototype Passport). Privacy/trust messaging preserved; no compliance claims.

## Guided Convert flow preserved

The Convert GDSN XML → Single XML guided flow (Upload → Mapping → Validate →
Export) is unchanged and still wired to the real converter/validator/reporter,
with all four real downloads (JSON-LD, mapping XLSX, validation JSON, unmapped
JSON).

## Preserved

- Converter logic, batch conversion behavior, and single-file output unchanged.
- Mapping YAML, catalog data, and Web Vocabulary snapshots unchanged.
- All seven workflows reachable; every governance warning preserved.
- No warnings suppressed; no mock data; no fabricated coverage/compliance.
- No GS1 ↔ Product Passport Crosswalk, SHACL execution, VC, or signed
  credentials. No online runtime fetching.

This release does not claim official GS1 validation.
