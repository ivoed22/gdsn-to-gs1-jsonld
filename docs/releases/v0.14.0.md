# v0.14.0 — App Modularization

## Overview

Version v0.14.0 splits the 2,500-line `app/streamlit_app.py` into a thin
router plus one module per workflow. There is **no user-facing behavior
change**: the converter, batch behavior, single-file output, mapping YAML,
mapping catalog, Web Vocabulary snapshots, and every workflow's rendered
output are all unchanged. All 197 tests, including the 24 AppTest
navigation/workflow tests, stay green.

It is the first step in the foundation-first roadmap sequenced in v0.13.5:
app modularization → visual smoke tests → Builder UX at scale.

## What moved

- **`app/workflow_shared.py`** — the route/workflow-mode registry
  (`ROUTES`, `WORKFLOW_MODES`), session-state helpers (`clear_results`,
  `clear_batch_results`, `clear_all_results`, `set_route`,
  `set_workflow_mode`), and the sidebar's backlog/WebVoc-metadata loaders.
  Shared by the router and multiple workflow modules.
- **`app/workflows/convert.py`** — Convert GDSN XML (single file + bulk ZIP).
- **`app/workflows/explore.py`** — Explore GS1 Web Vocabulary, including its
  cached dataset loader (also used by the prototype builder).
- **`app/workflows/candidates.py`** — Generate Mapping Candidates.
- **`app/workflows/standards.py`** — Standards Review.
- **`app/workflows/prototype.py`** — Create JSON-LD Prototype (the manual
  builder — the largest extracted module, including all field-rendering
  helpers).
- **`app/workflows/product_passport.py`** — Validate Product Passport
  Sources.
- **`app/workflows/product_passport_builder.py`** — Build Product Passport
  Prototype.
- **`app/streamlit_app.py`** — now a ~240-line router: page config, sidebar,
  the two-stage route/workflow navigation UI, and the dispatch that calls
  into the workflow modules above.

## Why

The single-file layout made it hard to read, test, or change one workflow
without touching the whole 2,500-line file. Splitting it is a pure
refactor that unblocks lower-risk feature work — starting with v0.15.0's
browser-based visual smoke tests, which need to target individual workflow
renders.

## How this stayed behavior-preserving

- Extracted one workflow at a time; ran the full test suite after each
  extraction (nine extraction steps, each landing on a green `197 passed`).
- `tests/test_streamlit_app.py` drives the app exclusively through
  `AppTest.from_file("app/streamlit_app.py")` and selects buttons by key
  (`route_<key>`, `workflow_mode_<key>`) or label, never by importing
  internal functions — so moving functions between files is invisible to
  the test suite by construction. The three attributes the tests do reach
  into directly (`REPOSITORY_ROOT`, `SRC_DIRECTORY`, `main`) stay on
  `app.streamlit_app`.

## Preserved

- Converter logic, batch behavior, and single-file output are unchanged.
- Mapping YAML, mapping catalog, and Web Vocabulary snapshots are unchanged.
- No warnings suppressed; no mock data; no fabricated coverage/compliance.
- No GS1 ↔ Product Passport Crosswalk, SHACL execution, VC, or signed
  credentials are built in this release.
