# v0.11.0 — Mapping Candidate Generator

## Overview

Version v0.11.0 adds the Mapping Candidate Generator, a deterministic,
offline tool that proposes possible GDSN/BMS/XPath source fields for GS1
Web Vocabulary properties.

This release is a review-support release.  It does not change converter logic,
mapping YAML, mapping catalog data, batch conversion, sample conversion
semantics, or Web Vocabulary snapshots.

Candidates are review support only.  No mappings are automatically accepted
or written.  This release does not claim official GS1 validation or production
compliance.

## What's New

### Mapping Candidate Generator

A new backend module
`src/gdsn_to_gs1_jsonld/mapping_candidate_generator.py` implements:

- Loading of all reference inputs (WebVoc properties CSV, GDSN attributes
  CSV, mapping catalog CSV, YAML mapping, standards backlog).
- Deterministic, offline candidate scoring using 11 positive signals and
  3 negative signals.
- Confidence classification (high/medium/low/review_required).
- Review status assignment (proposed/already_mapped/review_required/not_recommended).
- Report serialization to JSON, CSV, and XLSX.
- Candidate summary generation.

### Scoring Approach

Each (WebVoc property, GDSN attribute) pair is scored by accumulating signal
weights:

| Signal | Weight |
|---|---|
| existing_mapping_catalog_match | 0.90 |
| semantic_resource_urn_match | 0.80 |
| mapping_yaml_canonical_field_match | 0.65 |
| exact_property_name_match | 0.70 |
| label_attribute_token_overlap | 0.35 |
| quantity_uom_compatible | 0.20 |
| definition_comment_overlap | 0.20 |
| xpath_terminal_match | 0.25 |
| range_datatype_compatible | 0.15 |
| code_list_signal | 0.15 |
| standards_review_linked | 0.10 |

Negative signals: deleted_attribute_warning (-0.30), datatype_mismatch_warning
(-0.15), class_row_not_attribute (-0.20).

Scores are capped in [0.0, 1.0].

### CLI Command

New command: `gdsn-to-gs1-jsonld generate-mapping-candidates`

```bash
gdsn-to-gs1-jsonld generate-mapping-candidates \
  --webvoc-properties reference_data/normalized/webvoc_properties_1_17.csv \
  --gdsn-reference reference_data/normalized/gdsn_attributes_bms_xpath_3_1_36.csv \
  --catalog mapping_catalog/gdsn_to_gs1_web_vocabulary_mapping_catalog_v0_3_webvoc_validated.csv \
  --mapping mapping/mapping_v0_3.yaml \
  --standards-backlog docs/standards-decisions/standards_review_backlog.json \
  --output-dir mapping_candidate_reports/
```

Options: `--property`, `--limit-per-property`, `--min-confidence`,
`--include-low-confidence`, `--include-review-required`, `--format`.

### Streamlit Workflow

A fifth workflow card is added to the Streamlit app:

- Marker: MAP
- Title: Generate Mapping Candidates
- Description: Propose possible GDSN/BMS/XPath source fields for GS1 Web
  Vocabulary properties, with confidence and review reasons.
- Outcome: Review-only candidate report. No mappings are automatically
  accepted or written.

The workflow is displayed in a 3+2 grid layout (first row: XML, VOC, LD;
second row: SDR, MAP).

### Report Formats

- `mapping_candidates.json` — full candidate list as JSON array
- `mapping_candidates.csv` — flat CSV for spreadsheet review
- `mapping_candidates.xlsx` — Excel workbook (requires openpyxl)
- `mapping_candidates_summary.json` — aggregated counts
