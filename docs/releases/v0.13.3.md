# v0.13.3 — Guided Route Navigation

## Overview

Version v0.13.3 is a UI/UX navigation polish release. It simplifies the workflow
entry point with a calmer, goal-oriented two-stage flow: choose one of three
primary routes, then choose the tool within that route. No conversion,
validation, or reporting behaviour changes.

No new features. No mock data. No fabricated coverage. No compliance claims.
This release does not claim official GS1 validation or production compliance.

## What changed

### Two-stage guided route navigation

**Stage 1 — three primary routes** (large route cards, visually heavier than
child cards, with a clear active-state indicator and monospace markers):

- **Create GS1 JSON-LD** (`JSON-LD`) — Create GS1 Web Vocabulary JSON-LD from
  product XML or manual prototype input.
- **Vocabulary & Mapping** (`MAP`) — Review vocabulary, mappings, candidate
  sources and standards decisions.
- **Product Passport Bridge** (`PASS`) — Work with Product Passport sources and
  prototype passport output.

**Stage 2 — child workflows** for the selected route only (progressive
disclosure), under a context heading:

- Create GS1 JSON-LD → *Choose how to create JSON-LD*: Convert GDSN XML, Create
  JSON-LD Prototype.
- Vocabulary & Mapping → *Choose a review tool*: Explore GS1 Web Vocabulary,
  Generate Mapping Candidates, Standards Review.
- Product Passport Bridge → *Choose a Product Passport tool*: Validate Product
  Passport Sources, Build Product Passport Prototype.

### Behaviour

- Default route is **Create GS1 JSON-LD**; **Convert GDSN XML** is the default
  active workflow and its guided four-step flow renders as before.
- Selecting a route opens that route's first child as the active workflow — a
  standard master–detail pattern — **without** clearing conversion results.
- Clicking a child workflow opens the existing workflow; all existing render
  functions, session state, and result behaviour are reused unchanged.
- The route layer is a UI grouping only: every workflow key
  (`convert`, `prototype`, `explore`, `candidates`, `standards`,
  `product_passport`, `product_passport_builder`) is unchanged and reachable.

## Preserved from v0.13.2

- Sidebar workspace status/context, compact hero, wider workspace, and the
  labelled "Core conversion traceability" rail.
- Convert guided four-step flow (Upload → Mapping → Validate → Export) wired to
  the real converter/validator/reporter, with the four real downloads.

## Responsive note

Route cards use a three-column row that stacks on narrow viewports via
Streamlit's own column behaviour. Child rows size to the number of children in
the selected route (two or three), so no empty filler cards are shown.

## Preserved

- Converter logic, batch behavior, and single-file output are unchanged.
- Mapping YAML, catalog data, and Web Vocabulary snapshots are unchanged.
- All seven workflows reachable; governance warnings preserved.
- No warnings suppressed; no mock data; no fabricated coverage/compliance.
- No GS1 ↔ Product Passport Crosswalk, SHACL execution, VC, or signed
  credentials. No online runtime fetching.
