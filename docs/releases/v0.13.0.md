# v0.13.0 — Product Passport Builder

## Overview

Version v0.13.0 adds the **Product Passport Builder** in minimal-schema
prototype mode. It wraps GS1 Web Vocabulary JSON-LD into a prototype Product
Passport JSON-LD envelope and validates the built envelope against the committed
built-in minimal structural schema.

Prototype/reference only. Structural validation only. This release does **not**
claim official GS1 validation, **not** EU DPP regulatory compliance, and is
**not** production-ready.

## Why minimal-schema mode

The Product Passport source manifest still contains placeholder external DPP
schemas/contexts/examples (source URLs and checksums not yet filled). Therefore
v0.13.0 deliberately validates only against the committed built-in minimal
schema and labels all output as prototype/reference. No real DPP schema is
invented, downloaded, or claimed.

## What's new

### Backend — `product_passport_builder.py`

Deterministic, offline functions:

- input: `load_gs1_jsonld`, `normalize_gs1_jsonld_input`
- extraction: `extract_product_identifier`, `extract_gtin`,
  `extract_product_name`, `extract_product_description`, `extract_brand`,
  `extract_language_values`
- envelope: `build_minimal_product_passport` (alias
  `build_product_passport_envelope`)
- validation: `validate_built_product_passport` (reuses the v0.12.x validator in
  `product_passport_sources` — no duplicated logic)
- output: `product_passport_to_json_bytes`,
  `product_passport_report_bytes_json`, `build_product_passport_summary`,
  `write_product_passport_outputs`

Envelope fields include `@context`, `@type`, `@id`, `productPassportId`,
`passportType`, `passportVersion`, `status`, `prototypeNotice`,
`defaultLanguage`, `source` (sourceType/sourceFormat/sourceGtin/
sourceProductName), `validation` (validationMode/schema/note), and the embedded
source GS1 JSON-LD under `product` (toggleable). Default output is byte-stable;
`generatedAt` is omitted unless explicitly supplied.

### CLI — `build-product-passport`

```bash
gdsn-to-gs1-jsonld build-product-passport \
  --input product_passport/examples/gs1_product_for_passport_builder.jsonld \
  --schema product_passport/reference_sources/raw_public/schemas/dpp_minimal.schema.json \
  --output-dir product_passport/builder_outputs/
```

Options: `--passport-id`, `--default-language`,
`--include-source-gs1-jsonld / --no-include-source-gs1-jsonld`, `--format`.
Prints the prototype/reference warning, schema, validator mode, structural
validation status, and output paths. Exit 0 on success; non-zero only on tool
error.

### Streamlit — "Build Product Passport Prototype" (marker: PB)

Four tabs: Input GS1 JSON-LD, Builder Settings, Product Passport Output, and
Validation Report. The overview grid is now a 4+3 layout for seven workflows.
The workflow states clearly that it is prototype/reference, minimal-schema,
structural-validation-only, and not official GS1 validation, EU DPP compliance,
or production-ready. Placeholder external schemas are shown as unavailable.

### Report outputs

- `product_passport.jsonld`
- `product_passport_validation_report.json`
- `product_passport_summary.json`

## Limitations

- Minimal-schema mode only; no real DPP schema validation.
- No GS1 ↔ Product Passport Crosswalk (planned v0.14.0).
- No SHACL execution.
- No Verifiable Credentials / signed credentials.
- No online fetching.

## Relation to other features

- Builds on **v0.12.x Product Passport Source Validator** (reuses its validator
  and manifest concepts).
- Precedes the future **v0.14.0 GS1 ↔ Product Passport Crosswalk** and later
  SHACL / Digital Link / EPCIS / VC work.

## Preserved

- Converter logic, batch behavior, and single-file output are unchanged.
- Mapping YAML files, catalog data, and Web Vocabulary snapshots are unchanged.
- No warnings were suppressed.
- No Crosswalk, SHACL execution, VC, or signed credentials were created.
- No online runtime fetching was added.

## No official validation

This release does not claim official GS1 validation.
This release does not claim production or EU DPP regulatory compliance.
