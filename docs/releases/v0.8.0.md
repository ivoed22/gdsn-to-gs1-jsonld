# v0.8.0 — Workflow Modes and Bulk XML Upload

## Overview

Version v0.8.0 restructures the Streamlit app into workflow modes and adds a
safe bulk ZIP upload path for converting multiple GDSN XML files.

The existing single XML workflow remains available under `Convert GDSN XML` and
keeps the same converter path, output content, download names, and supporting
reports.

## Added

- Added a top-level Streamlit workflow selector:
  - `Convert GDSN XML`
  - `Explore GS1 Web Vocabulary`
  - `Standards Review`
- Added `Single XML` and `Bulk ZIP` tabs inside `Convert GDSN XML`.
- Added a reusable `batch_converter` backend for safe ZIP processing.
- Added `gdsn-to-gs1-jsonld convert-batch`.
- Added batch export ZIPs with `batch_summary.json`,
  `batch_summary.xlsx`, product JSON-LD files, per-file reports, and
  per-file error JSON files.
- Added a Web Vocabulary Explorer placeholder for a later dedicated Explorer
  release.
- Added a compact read-only Standards Review mode using existing backlog data.
- Added batch conversion regression tests and Streamlit workflow mode tests.
- Added a UI quality checklist covering workflow visibility, single/bulk
  conversion, readable standards review, Explorer placeholder visibility, and
  avoidance of generic/template-looking UI regressions.
- Applied final UI audit polish to reduce hero/rail redundancy, simplify
  workflow card states, and keep the traceability rail as the main evidence
  story.

## Preserved

- Single-file converter output is unchanged.
- Mapping YAML files are unchanged.
- Catalog data is unchanged.
- Web Vocabulary snapshots are unchanged.
- No warnings were suppressed.
- No semantic mappings were changed.
- Existing single XML Streamlit behavior remains available in the `Single XML`
  tab.
