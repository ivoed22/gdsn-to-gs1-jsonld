# v0.5.0 — Real-world GDSN Sample Robustness

## Overview

Version v0.5.0 improves robustness, diagnostics, and sample coverage for
realistic GDSN-like XML inputs.

This release does not add new business mapping scope. Instead, it strengthens
confidence in the existing converter pipeline by adding a sample corpus, batch
sample conversion, improved context in diagnostics, and regression tests
across multiple realistic input variants.

## Added

### Sample corpus

Added a structured sample corpus under `examples/input/samples/`:

- `minimal_product.xml`
- `food_product_full.xml`
- `certified_product_with_documents.xml`
- `partially_mapped_product.xml`

All samples are synthetic and use fake GTINs and `example.com` URLs only.

### `convert-samples` CLI command

Added:

```bash
gdsn-to-gs1-jsonld convert-samples
```

This command converts all XML files in a sample directory and produces:

- JSON-LD output
- validation reports
- unmapped field reports
- sample-level summary reports

### Sample conversion summary reports

Added generated sample summary outputs:

- `sample_conversion_summary.json`
- `sample_conversion_summary.xlsx`

The summary includes:

- sample file
- detected GTIN
- conversion success
- output file paths
- validation status
- validation error and warning counts
- unmapped element counts
- mapped field counts
- notes

### Improved diagnostics

Improved diagnostic context for repeated XML structures, including:

- language-specific values
- nutrient details
- allergen details
- certification details
- referenced document information

This makes unmapped reports easier to interpret when realistic XML contains
repeated or partially mapped structures.

## Changed

- Improved sample-level conversion handling.
- Improved conversion diagnostics.
- Added regression coverage for multiple sample variants.
- Updated documentation for real-world sample testing.
- Preserved existing v0.1.0, v0.2.0, and v0.3.0 mapping behavior.
- Mapping YAML files remain unchanged.

## Known limitations

This release does not implement:

- full GDSN XSD validation
- new business mapping scope
- Verifiable Credential verification
- DCAT/DPROD modelling
- Digital Link resolver calls
- certificate validation
- full Digital Product Passport modelling

The sample files are synthetic and intended for testing only. Real GDSN XML
files may still require mapping adaptation depending on module structure,
namespaces, and implementation-specific content.

## Notes for users

The unmapped report is a diagnostic aid. It does not mean that the source XML
is invalid.

The sample conversion command helps users test converter behavior across
multiple XML inputs and inspect conversion coverage before using the tool with
real GDSN data.
