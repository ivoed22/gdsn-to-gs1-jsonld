# v0.5.0 — Real-world GDSN Sample Robustness

Version 0.5.0 improves testing robustness and diagnostics for realistic
GDSN-like XML without expanding business mapping scope.

## Added

- synthetic minimal, food, certification/document, and partially mapped XML
  samples
- `convert-samples` CLI command
- per-sample JSON-LD, validation, mapping, and unmapped reports
- JSON and Excel sample conversion summaries
- explicit mapping, XML parsing, validation/conversion, and output-writing
  failure diagnostics
- language and repeated-structure discriminator context in unmapped reports
- sample corpus and diagnostic regression tests
- guidance for testing private real-world GDSN XML locally

## Preserved

- v0.1.0 JSON-LD output compatibility
- v0.2.0 JSON-LD output compatibility
- v0.3.0 JSON-LD output compatibility
- existing mapping YAML files and business mapping scope
- v0.4.0 mapping catalog quality checks

## Limitations

The included files are synthetic test data and use fake GTINs and
`example.com` URLs. The converter does not perform full GDSN XSD validation.
Unmapped reports are diagnostic and do not prove that source XML is invalid.

This release does not implement DPP expansion, Verifiable Credentials,
DCAT/DPROD, resolver calls, URL dereferencing, or certificate verification.
