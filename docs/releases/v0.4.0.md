# v0.4.0 — Mapping Catalog Driven Quality Checks

Version 0.4.0 adds governance checks that compare executable YAML mapping
profiles with the GDSN-to-GS1 Web Vocabulary mapping catalog.

## Added

- reusable catalog loading and validation
- required-column, mapping-status, confidence, BMS ID, and row-completeness
  checks
- Web Vocabulary review diagnostics
- YAML extraction for simple fields, object mappings, and child fields
- YAML/catalog canonical-field and JSON-LD-property comparison
- `check-catalog` and `check-mapping` CLI commands
- strict mode for warning-as-failure workflows
- structured JSON and multi-sheet Excel reports
- tests and mapping quality documentation

## Preserved

- v0.1.0 converter output compatibility
- v0.2.0 converter output compatibility
- v0.3.0 converter output compatibility
- `mapping_mvp.yaml`, `mapping_v0_2.yaml`, and `mapping_v0_3.yaml`
- converter logic shared by CLI and Streamlit

## Reports

Quality reports include errors, warnings, informational findings, YAML and
catalog coverage, missing fields, experimental mappings, review items, and Web
Vocabulary issues. Warnings are non-failing unless `--strict` is used.

## Limitations

This release does not add new GDSN business mappings and does not generate YAML
from the catalog. It does not implement DPP expansion, Verifiable Credentials,
DCAT/DPROD, resolver calls, certificate verification, URL dereferencing, or
full GDSN XSD validation.
