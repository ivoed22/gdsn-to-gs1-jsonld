# v0.9.0 — Web Vocabulary Explorer

## Overview

Version v0.9.0 turns the `Explore GS1 Web Vocabulary` placeholder into a real
read-only Explorer for the local GS1 Web Vocabulary snapshot.

The Explorer helps users browse local classes and properties, filter vocabulary
terms by review group and coverage status, and compare Web Vocabulary terms
with the existing GDSN mapping catalog evidence and open SDR governance topics.

## Added

- Added `src/gdsn_to_gs1_jsonld/webvoc_explorer.py`.
- Added local WebVoc JSON-LD loading and extraction for classes and properties.
- Preserved labels, comments, domain, range, `subPropertyOf`, type, term
  status, full IRI, and link-type indicators where available.
- Added pragmatic property grouping for GS1/GDSN mapping review.
- Added mapping coverage status classification:
  - `mapped`
  - `high_confidence`
  - `candidate`
  - `experimental`
  - `standards_review_required`
  - `schema_org_fallback`
  - `unmapped`
  - `unknown`
- Added BMS/XPath evidence from the existing mapping catalog.
- Added SDR/governance indicators from the existing standards review backlog.
- Replaced the Streamlit Explorer placeholder with:
  - WebVoc version, class count, property count, mapped property count, and
    standards-review property count;
  - group, domain, coverage, search, mapped-only, and standards-review filters;
  - a read-only property table;
  - a property detail expander with term metadata, mapping evidence, and SDR
    notes;
  - a clearly labelled `Manual JSON-LD prototype — planned` panel.
- Added `gdsn-to-gs1-jsonld export-webvoc-explorer`.
- Added Explorer JSON, CSV, summary JSON, and summary XLSX exports.
- Added documentation in `docs/webvoc-explorer.md`.
- Added backend, CLI, helper, and Streamlit regression tests.

## Notes

The Explorer is intentionally read-only in v0.9.0. Manual JSON-LD authoring is
documented as planned future functionality because manual output would not be
GDSN/BMS/XPath traceable unless connected to governed mapping evidence.
