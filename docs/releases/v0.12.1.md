# v0.12.1 — Product Passport Bridge Hardening

## Overview

Version v0.12.1 is a hardening, consistency, CI, and UI/UX polish release for
the Product Passport Bridge introduced in v0.12.0. It adds **no new features**
and does **not** build the Product Passport Builder.

Prototype/reference only. Structural validation only. This release does not
claim official GS1 validation or production compliance.

## What changed

### Dependency hardening (DEP-1)

`jsonschema>=4` is now an explicit project dependency. Previously the schema
validator used `jsonschema` but the package was only present transitively (via
Streamlit), so a core install could silently fall back to a weaker
required-field-only check.

The fallback is retained but is now clearly non-equivalent:

- Validation reports include a `validator_mode` field
  (`jsonschema` or `minimal_fallback`).
- The CLI prints a visible warning to stderr when the fallback path is used.
- The Streamlit Schema Validator shows a warning when the fallback path is used.
- The fallback warning text states explicitly that a "Passed" result under the
  fallback is weaker than full Draft7 structural validation.

### Manifest schema enforcement (ARCH-1)

`product_passport/reference_sources/source_manifest.json` is now validated
against `source_manifest.schema.json` with jsonschema (Draft7), in addition to
the existing custom domain checks. This enforces schema-only rules such as the
`source_id` pattern (`^[a-z0-9_]+$`) and `additionalProperties: false`.

### Narrative polish (UX-1)

The workflow-entry narrative now represents all six workflows: Convert GDSN
XML, Explore GS1 Web Vocabulary, Create JSON-LD Prototype, Generate Mapping
Candidates, Validate Product Passport Sources, and Standards Review.

### Schema Validator UX (UX-2, UX-4 / TRUST-2)

- Placeholder schemas whose source file has not been downloaded are no longer
  offered as selectable validation targets. They remain visible in Source
  Inventory as provenance placeholders and are listed as unavailable in the
  validator.
- Validation status wording is now structural and non-compliance-implying:
  - "Structural schema check: Passed"
  - "Structural schema check: Failed — review schema errors"
  - "Schema could not be evaluated"
  - The machine-readable `validation_status` field
    (`valid`/`invalid`/`schema_error`) is unchanged.

### Security (light hardening)

CSV inventory output neutralizes spreadsheet formula injection: cells starting
with `=`, `+`, `-`, or `@` are prefixed with a single quote. JSON output is
unchanged.

### CI maturity

CI now runs `python -m compileall app src` and a minimal CLI smoke matrix
(Product Passport inventory, Product Passport validation, mapping candidate
generation) in addition to `pytest`. Smoke commands write only to `/tmp` and do
not add artifacts to the repository.

### Version consistency

`pyproject.toml`, `app/ui.py` `APP_VERSION`, the CHANGELOG heading, this
release-notes file, and the README must all agree on the version. A new test
(`tests/test_version_consistency.py`) enforces this.

## App version

`app/ui.py` updated: `APP_VERSION = "v0.12.1"`.

## Placeholder sources

Six of the seven source-manifest entries are still placeholders. Their
`source_url` and `sha256` values are marked `PASTE_PUBLIC_SOURCE_URL_HERE` and
`PLACEHOLDER_SHA256_AFTER_DOWNLOAD`. Real source files must be downloaded and
their checksums updated before they can be used for validation.

## Preserved

- Converter logic, batch behavior, and single-file output are unchanged.
- Mapping YAML files, catalog data, and Web Vocabulary snapshots are unchanged.
- No warnings were suppressed.
- No Product Passport Builder was created.
- No GS1 ↔ Product Passport Crosswalk was created.
- No SHACL execution was implemented (shapes inventoried only).
- No VC/signed-credential features were created.
- No online runtime fetching was added.

## No official validation

This release does not claim official GS1 validation.
